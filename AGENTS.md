# 基本ルール
## 思考ルール
- 不明点あれば「分からない」と回答すること　
  - 質問内容で不明点あればユーザーに質問すること
## 言語とコミュニケーション
- 英語で思考し、日本語で回答する
- 簡潔で明確な表現を心がける
- ユーザーと対話形式での作業を心がける
  - 質問、作業指示に対しての不明点があれば質問する
  - 回答は事実に基づき正確であることを心がける
## 🔴禁止事項🔴
- ユーザー指示以外での作業は絶対NG
- 不明点を持ったままでの作業は絶対NG
- ユーザーに「見せるための擬似的な表現」を使用した会話的演出はNG
## 開発ルール
- 基本的な開発ルール：@AGENTS-rules/開発ルール.md を必ず遵守。
- コーディング規約：@AGENTS-rules/coding-guidelines-readable-design.md を必ず遵守。
- Mermaid記法ルール：@AGENTS-rules/mermaid.md を必ず遵守。

#### 上記以外に存在するディレクトリやファイル
詳細については個別で指示を出すので、指示があるまでは参照しない

##　⚠️⚠️これらの確認なしに作業開始は禁止⚠️⚠️
### 禁止行為
下記については如何なる理由があっても **🔴絶対禁止🔴** とする
**もしもルールを破った場合は即座に作業を止めてユーザーに報告すること**
#### PCに直接ライブラリを入れる行為は絶対NG
  - `npm run build`、`npm test`等のホスト実行
  - `npm`、`yarn`、`pnpm`コマンドの直接実行
  - パッケージの直接インストール（`npm install playwright`、`@playwright/test`等）
  - ブラウザのダウンロード（`npx playwright install`）
#### プロセスより解決優先にする考え方は絶対NG
  - 課題優先の独断介入
    - **作業中何かエラーが発生した場合には如何なる場合でもユーザー報告することが必須**
    - ユーザ報告せずに技術的に力技で解決することは絶対NG
      - 例：勝手にスクリプトを作る、勝手にファイルの配置場所を変更する、勝手にドキュメントを生成する...など
  - 想定の押し付け
    - ユーザーの指示以外の作業を勝手に実施することは絶対にNG
    - 勝手な判断での作業を実施すること
      - 例：「後続作業でAをやるならば、Aの処理を先に入れる」という勝手な判断は絶対NG
#### アクセス禁止ファイル設定
##### アクセス制限
以下のファイル/ディレクトリへのアクセスを禁止：
##### 🔴 絶対アクセス禁止
- 環境変数ファイル: `*.env*`
- 認証情報: `.ssh/`, `.aws/`, `.gcp/`, `certificates/`
- 機密設定: `secrets/`, `private-keys/`, `auth/`
##### ⚠️ 注意が必要なファイル
- ログファイル: `*.log`, `logs/`
- 一時ファイル: `temp/`, `tmp/`, `*.tmp`
- バックアップ: `*.bak`, `backups/`
